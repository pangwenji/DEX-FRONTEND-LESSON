# 第 2 课：流动性池列表

> 建议课时：45～60 分钟｜对应页面：`/pools`

## 学习目标

- 理解交易对、费率、TVL、交易量和池地址；
- 说明为什么前端通常同时读取链上数据和索引数据；
- 使用 API 与 Hook 拉取池列表，并处理加载、空数据和异常状态；
- 从池列表导航到动态详情页。

## 业务模型

流动性池保存两种代币及其流动性。列表页用于“发现池子”，常见指标如下：

| 指标 | 含义 | 数据来源示例 |
| --- | --- | --- |
| Token0 / Token1 | 交易对 | 链上合约 |
| Fee | 每次交换的手续费档位 | 链上合约 |
| TVL | 池中资产的估算总价值 | 链上余额 + 价格 |
| 24h Volume | 24 小时成交额 | 事件索引 / 数据库 |
| Pool Address | 池的唯一地址 | Factory / PoolManager |

> TVL 与交易量不是同一个概念；前者是存量，后者是时间区间内的流量。

## 数据流

```text
Pools 页面 → usePools → GET /api/pools
                         ├─ 链上读取池子状态
                         └─ 数据库读取事件与聚合值
              → 格式化 → 卡片列表 → /pools/[address]
```

## 实操任务

1. 启动 `lesson1.2/code`，打开 `/pools`。
2. 在浏览器 Network 面板查看 `/api/pools` 的响应结构。
3. 找到页面的 loading、error、empty 和 success 四种渲染分支。
4. 点击池卡片，观察地址如何成为动态路由参数。
5. 为池地址增加缩略显示，但复制时保留完整地址。

## 前端实现要点

- 不要用数组下标作为池卡片的 key，优先使用池地址。
- 地址进入查询前应先做格式校验；展示时可缩略，计算时不可截断。
- 金额格式化要明确 decimals，并对大数使用 `bigint`/专用工具。
- 允许单项指标缺失，避免一个统计值失败导致整个列表不可用。
- 自动刷新时保留旧数据，可减少页面闪烁。

## 课堂练习

给列表增加三个筛选能力：按代币符号搜索、按 TVL 排序、按费率筛选。思考筛选应该在前端还是接口端完成，以及数据量变大后的选择。

## 常见问题

- **列表为空**：检查网络、合约地址、扫描器进度和接口响应。
- **TVL 显示 NaN**：检查空值、代币 decimals 与字符串转数值过程。
- **同一池重复出现**：用规范化后的小写地址去重。
- **刷新后数据抖动**：查询期间保留上一轮数据，并区分首次加载与后台刷新。
